# Чеклист: Создание питч-сайта

Пошаговая инструкция для запуска нового проекта.

---

## Этап 1: Подготовка материалов

### Контент (обязательно)
- [ ] **Название проекта** — короткое, запоминающееся
- [ ] **Слоган** — одна сильная фраза (крючок)
- [ ] **Формат** — жанр, количество серий, хронометраж
- [ ] **Логлайн** — 1-2 предложения
- [ ] **Синопсис** — 3-5 абзацев
- [ ] **Персонажи** — имя, возраст, типаж, описание, арка (3-5 героев)
- [ ] **Режиссёрский стейтмент** — личное высказывание
- [ ] **Контакты** — имя, роль, email, телефон

### Контент (рекомендуется)
- [ ] PDF сценария пилота
- [ ] PDF синопсиса всех серий
- [ ] Визуальные референсы (2-4 фильма/сериала)
- [ ] Цветовая концепция
- [ ] Музыкальные референсы
- [ ] Подход к камере
- [ ] Преимущества формата
- [ ] Описание аудитории

### Изображения (обязательно)
- [ ] **Hero фон** — атмосферное изображение (1920×1080)
- [ ] **Фото персонажей** — актёры или референсы (600×800)

### Изображения (рекомендуется)
- [ ] Постеры референсов (400×600)
- [ ] Референсы локаций
- [ ] Мудборды (цвет, костюмы)
- [ ] Раскадровка
- [ ] Фоны для секций

## Этап 2: Настройка проекта

### Создание структуры
```bash
# Создать папку проекта
mkdir my-pitch-site
cd my-pitch-site

# Скопировать шаблоны
cp /path/to/template/* .

# Создать папки для медиа
mkdir -p images materials
mkdir -p images/characters images/references images/locations
```

### Файловая структура
```
my-pitch-site/
├── index.html
├── styles.css
├── script.js
├── images/
│   ├── hero-bg.jpg
│   ├── characters/
│   ├── references/
│   └── locations/
└── materials/
    ├── screenplay.pdf
    └── synopsis.pdf
```

#### Ритм и эмоция

- [ ] Добавить жанры и их функции

#### Режиссёрское видение
- [ ] Заполнить стейтмент
- [ ] Описать подход к актёрам
- [ ] Описать подход к камере
- [ ] Добавить локации (если есть)

#### Производство
- [ ] Указать хронометраж
- [ ] Описать структуру
- [ ] Перечислить преимущества
- [ ] Описать аудиторию

#### Контакты
- [ ] Заполнить контактную информацию

#### Футер
- [ ] Обновить год и название

### styles.css

- [ ] Добавить фон Hero:
```css
.hero {
    background-image: url('images/hero-bg.jpg');
}
```

- [ ] Настроить фоны секций (опционально):
```css

    background: linear-gradient(...), url('images/story-bg.jpg');
}
```

### script.js

- [ ] Настроить слайдеры локаций (если есть):
```javascript

    location1: {
        title: 'Название',
        images: ['images/locations/1.jpg', ...]
    }
};
```

- [ ] Настроить видео (если есть):
```javascript
const videos = {
    video1: {
        title: 'Название',
        url: 'https://www.youtube.com/embed/VIDEO_ID'
    }
};
```

## Этап 5: Публикация

### GitHub Pages

git init
git add .

## Быстрый старт (копипаста)

### 1. Минимальный набор для запуска
```
- Название + слоган + формат
- Логлайн
- Синопсис (хотя бы 2 абзаца)
- 2-3 персонажа
- Режиссёрский стейтмент
- Контакты
- Hero фон
```

### 2. Команды для локального просмотра
```bash
# Python 3
python3 -m http.server 8080

# Или Node.js
npx serve
```

### 3. Шаблон коммита
```bash
git add .
git commit -m "feat: add [section] content"
git push
```

## Частые ошибки

1. **Забыли обновить версию CSS/JS**
   - Увеличить `?v=X` при каждом изменении

2. **Изображения не отображаются**
   - Проверить пути (регистр важен!)
   - Проверить расширения файлов

3. **Слайдер не работает**
   - Проверить ключи в `locationSliders`
   - Проверить onclick в HTML

4. **Мобильное меню не закрывается**
   - Проверить класс `.active` в CSS

5. **PDF не открывается**
   - Проверить путь в `materials/`
   - Убедиться, что файл загружен
