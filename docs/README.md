## Шаблонные файлы

В папке `template/` находятся чистые файлы с плейсхолдерами:

```
template/
├── index.html    # HTML-шаблон с комментариями
├── styles.css    # CSS со всеми стилями
└── script.js     # JavaScript с настраиваемыми компонентами
```

### Как использовать шаблон

1. Скопировать файлы из `template/` в новый проект
2. Заменить плейсхолдеры (`<!-- ... -->`) на свой контент
3. Добавить изображения в папку `images/`
4. Настроить слайдеры в `script.js`
5. Опубликовать на GitHub Pages

## Быстрый старт

### 1. Создать проект
```bash
mkdir my-pitch && cd my-pitch
cp -r /path/to/template/* .
mkdir -p images materials
```

### 2. Минимальный контент
- Название + слоган + формат
- Логлайн (1-2 предложения)
- Синопсис (3-5 абзацев)
- 2-3 персонажа с описанием
- Режиссёрский стейтмент
- Контакты

### 3. Минимальные изображения
- Hero фон (1920×1080)
- Фото персонажей (600×800)

### 4. Локальный просмотр
```bash
python3 -m http.server 8080
# Открыть http://localhost:8080
```

### 5. Публикация
```bash
git init && git add . && git commit -m "Initial"
git remote add origin https://github.com/user/repo.git
git push -u origin main
# Включить GitHub Pages в настройках репозитория
```

## Рекомендации

### DO
- Показывать, а не рассказывать (больше визуала)
- Эмоциональный, но не пафосный язык
- Конкретика вместо абстракций
- Сравнения с известными проектами
- Личное отношение автора

### DON'T
- Длинные тексты без структуры
- Абстрактные описания ("уникальный", "невероятный")
- Слишком много информации сразу
- Технические детали без контекста
- Обещания без подтверждения
