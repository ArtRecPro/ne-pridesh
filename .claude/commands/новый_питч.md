# Создание нового питч-сайта

## Аргумент
$ARGUMENTS — название нового проекта

## Инструкции

1. **Создай папку для нового проекта:**
   ```
   /root/[название_проекта]/
   ```

2. **Скопируй шаблонные файлы:**
   - Из `/root/не_придешь/template/index.html`
   - Из `/root/не_придешь/template/styles.css`
   - Из `/root/не_придешь/template/script.js`

3. **Создай структуру папок:**
   ```
   /root/[название]/
   ├── index.html
   ├── styles.css
   ├── script.js
   ├── images/
   │   ├── characters/
   │   ├── references/
   │   └── locations/
   └── materials/
   ```

4. **Спроси у пользователя базовую информацию:**
   - Полное название проекта
   - Слоган / крючок
   - Жанр и формат (X серий × Y минут)
   - Логлайн (1-2 предложения)

5. **Заполни Hero секцию** в index.html с полученными данными

6. **Запусти локальный сервер** для просмотра:
   ```bash
   cd /root/[название]/ && python3 -m http.server 8080
   ```

7. **Покажи пользователю** ссылку на локальный просмотр и спроси, какой раздел заполнять дальше

## Документация
При необходимости обращайся к документации:
- `/root/не_придешь/docs/01-DESIGN-SYSTEM.md` — дизайн-система
- `/root/не_придешь/docs/02-PITCH-STRUCTURE.md` — структура контента
- `/root/не_придешь/docs/03-TECHNICAL.md` — технические паттерны
- `/root/не_придешь/docs/04-CHECKLIST.md` — чеклист
